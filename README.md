# Scrabble Helper (RU/EN)

Небольшое офлайн-friendly веб-приложение для подбора слов в формате «Скрэббл / Эрудит».

- Поддерживает русский и английский словари.
- Фильтрует слова по набору букв.
- Поддерживает маску (`_` — одна любая буква, `*` — любое количество букв).
- Сортирует результаты по очкам.

## Требования

Нужен только статический HTTP-сервер (просто открыть `index.html` из файловой системы нельзя, т.к. словари подгружаются через `fetch`).

## Локальный запуск

Из корня проекта (`/workspace/scrabble-helper`) запустите любой удобный сервер.

### Вариант 1: Python

```bash
python3 -m http.server 8080
```

Откройте в браузере: `http://localhost:8080`

### Вариант 2: Node.js (`serve`)

```bash
npx serve .
```

## Публикация на GitHub Pages

Да — приложение полностью статическое, поэтому его можно без проблем выложить на GitHub Pages.

### Способ A (самый простой): Deploy from branch

1. Запушьте репозиторий на GitHub.
2. В репозитории откройте **Settings → Pages**.
3. В разделе **Build and deployment** выберите:
   - **Source:** `Deploy from a branch`
   - **Branch:** `main` (или ваша рабочая ветка) и папка `/ (root)`
4. Сохраните настройки и дождитесь публикации.

После этого сайт будет доступен по адресу вида:

`https://<username>.github.io/<repo-name>/`

### Способ B: GitHub Actions

Если хотите отдельный процесс деплоя, можно использовать стандартный workflow `pages`.

Важно: в этом проекте все пути уже относительные (`styles.css`, `app.js`, `data/...`), поэтому приложение корректно работает из подпапки репозитория на Pages без дополнительной настройки `base`.
